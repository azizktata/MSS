# MMS — Project Design Brief

Source of truth for product content, structure, and data: `Design-doc.md`. This file holds the **Design Context** required by the `impeccable` skill (and `bolder`, which depends on it). Read both before any UI work.

## Design Context

### Users

Industrial procurement decision-makers in B2B bulk silica/quartz sourcing, export-oriented across the Mediterranean / North Africa basin (selling into Europe & MENA):

- **Plant engineers & QC / quality-control officers** — trust empirical spec tables (SiO₂, Fe₂O₃ limits, sieve analysis, COA) over marketing copy.
- **Procurement directors & volume category managers** — anxious about bulk-shipping timelines, tonnage reliability, and port-to-port risk.
- **Application specialists & R&D directors** — need confidence in chemical purity for premium formulations (glass, foundry, filtration, ceramics).

Context of use: evaluating a supplier for large-scale financial/logistical commitment. They commit on three empirical vectors — **chemical consistency, supply-chain predictability, application expertise** — not brand storytelling. The site must communicate all three within the first 5 seconds.

### Brand Personality

- **Three words:** credible, specialist, agile.
- **Voice/tone:** precise, technical, plain-spoken, confident. Lets auditable data do the selling. No hype, no fluff.
- **Emotional goal:** **"agile expert partner."** Not a faceless global major (Sibelco/Covia), not an unverified marketplace broker (Alibaba/EC21) — the responsive specialist who understands *your* application's technical thresholds. Closing the structural credibility gap is the whole job.

### Aesthetic Direction

This product **is** a variant toggle. Three first-class, fully distinct design philosophies switch via a sticky engineering control bar; **identical factual data, entirely different visual shells.** No single variant is primary — each must be production-grade and self-consistent. See `Design-doc.md` §4 for per-variant specs.

- **`/v1` Industrial Spec Engine** — raw utility. Monospace-dominant labels/metrics, sharp `0px` corners, 50/50 data-frame hero, low-opacity `1px` rule lines, processing-plant imagery.
- **`/v2` Deep Basin Carrier** — maritime scale. Cinematic full-bleed bulk-carrier hero, heavy editorial grotesque display type, generous vertical padding (`7rem 4rem`), asymmetric shipping-lane columns.
- **`/v3` Material Monolith** — premium lab/origin. Macro quartz under studio light, geometric sans with expansive letter-spacing on labels, bright mineral-stone surfaces, microscopic `3px` radii, tight padding.

**Theme — NO DARK SITES.** All three variants are light-themed. The graphite tone `--bg-3` (`oklch(25% 0.02 200)`) is reserved for accents only: hero photo overlays/protect-zones, sticky bars, footers, and inverted spec strips — never a full-page background. Page surfaces use `--bg` / `--bg-2`. Variants diverge **maximally** (type, layout, density, palette *weighting*) while staying one recognizable brand.

**Anti-references (explicitly NOT this):**
- Generic corporate template — no stock handshake photos, centered heroes, repeated identical card grids, or rounded-rectangle-with-drop-shadow everything.
- Flashy SaaS startup — no purple/cyan gradients, glassmorphism, gradient text, playful illustrations, or hero big-number metric layouts.
- Alibaba/EC21 marketplace — no cluttered listing-board look, trust badges as decoration, or generic-broker clutter.

### Locked Palette (user-tested — fixed brand truth)

The user tested this OKLCH palette on real sites and approved it. It is **fixed across all three variants** — do not substitute hues. Variants re-weight *which* tokens dominate; they do not change the tokens. Tint neutrals stay on hue 75 (warm quartz) for surfaces and hue 200 (cool graphite) for ink/lines; the accent is a single warm terracotta.

```css
--bg:          oklch(95% 0.01  75);    /* primary warm-quartz surface */
--bg-2:        oklch(92% 0.014 75);    /* secondary surface / alternating sections */
--bg-3:        oklch(25% 0.02  200);   /* graphite — ACCENT ONLY (overlays, bars, footer); never full-page bg */
--ink:         oklch(22% 0.02  200);   /* primary text */
--ink-soft:    oklch(36% 0.022 200);   /* secondary text */
--muted:       oklch(50% 0.018 200);   /* tertiary / captions */
--line:        oklch(22% 0.02  200 / 0.16);   /* hairline rules, grid gaps */
--line-strong: oklch(22% 0.02  200 / 0.48);   /* emphasized borders */
--accent:      oklch(55% 0.13  35);    /* terracotta — CTA / focus / rare emphasis (the 10%) */
--accent-ink:  oklch(96% 0.01  75);    /* text on accent */
--radius:      2px;
```

**Fonts — open per variant (user deferred font choice to me).** The user's examples reach for Space Grotesk + Inter; both are on the impeccable reject list, so per the user's decision I choose distinctive fonts *per variant* instead, keeping a stable mono for numerics where it earns its place. Pick three genuinely different display faces (one per variant) and a refined body face; vary numeric/tabular treatment too. Document the final picks here once chosen.

### Layout language (from user-approved examples)

These patterns are confirmed-liked and should recur (adapted, not copy-pasted, per variant):

- **Wide centered container:** `max-w-[1480px]` with fluid gutters `px-5 md:px-10 lg:px-14`; generous vertical section padding (`py-14`–`py-16`).
- **12-column asymmetric grids** (e.g. `lg:col-span-4 / lg:col-span-8`, `5/7`) — never symmetric 50/50 except where a variant calls for it.
- **Eyebrow + display H2 + measure paragraph** section headers, numbered (`I. The Corridor`, `II. The Material`).
- **Roman-numeral / coded section labels** and `.num` (tabular/mono) treatment for all metrics, vessel IDs, tonnages, ETAs, oxide values.
- **Full-bleed hero** with `protect-zone` gradient overlay over photo + a blurred graphite stat strip pinned to the bottom (vessel/loading/route/ETA style live-data row).
- **Spec tables** (`.spec`) for chemical assay + sieve matrix, horizontally scrollable on mobile (`scroll-x`).
- **`photo-grain` image masks** with `object-fit` cover; hover scale `1.03` inside `overflow-hidden` under 280ms exponential ease.
- **Grid-gap "hairline" card grids:** `gap-px` over a `--line` background so cards are separated by 1px rules, not shadows.

**Reconcile with impeccable bans:** the example's `borderLeft: 2px solid var(--accent)` on packaging cards is the banned side-stripe pattern — replace it with a full 1px `--line-strong` border, a `--bg-2` tint, or a leading `.num` value instead. No drop-shadow rounded rectangles; `--radius` stays at 2px (sharp). Accent terracotta is the 10% — reserve it for CTAs, focus rings, and rare emphasis.

### Design Principles

1. **Data is the hero.** Every section anchors on real, auditable specifications and a dominant thematic photo — never abstract marketing blocks. Use the exact data matrix in `Design-doc.md` §5.
2. **Three shells, one truth.** The variant toggle must produce genuinely different aesthetics over byte-identical content. Build shared data, swap presentation.
3. **Credibility over flash.** Distinctive and confident, never gimmicky. Pass the AI-slop test: someone should ask "how was this made?", not "which AI made this?"
4. **Engineered interaction.** Eight explicit states on every clickable element; <80ms initial feedback (`--duration-snappy: 120ms`, snappier sub-phase); `prefers-reduced-motion: reduce` collapses spatial animation to opacity crossfades; OKLCH throughout.
5. **Bilingual by construction.** EN primary, FR secondary (export market) — structure content so the i18n switch is first-class, not bolted on.
